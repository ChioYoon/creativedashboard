## ✅ 완료 내용

### 1. CSV 컬럼 매핑 수정
**파일**: `js/pipeline-engine.js`
- `plNormalizeColumns()` 함수 개선
- **문제점**: BNR/VID 행에서 '파일명' 값이 '소재명'을 덮어쓰는 로직
- **해결**: 두 컬럼을 독립적으로 보존하도록 수정
```javascript
// 이전: 파일명 → 소재명 복사 (덮어쓰기)
if (hasFilename && ['BNR', 'VID'].includes(currentType)) {
  newR['소재명'] = newR['파일명'].trim();
}

// 이후: 두 컬럼 모두 보존
if (newR['파일명']) newR['파일명'] = String(newR['파일명']).trim();
if (newR['소재명']) newR['소재명'] = String(newR['소재명']).trim();
```

### 2. 집계 로직 강화
**파일**: `js/analysis-engine-v2.js`
- `aggregateCreativeData()` 함수 개선
- **TXT 타입 자동 제외**: BNR/VID만 분석 대상
- **groupBy 파라미터 검증**: 컬럼 존재 여부 확인 및 경고 메시지
- **디버깅 로그 추가**: 집계 과정 상세 로그 출력
```javascript
// BNR/VID만 필터링
const bnrVidRows = rows.filter(r => 
  ['BNR', 'VID'].includes((r['유형'] || '').toUpperCase())
);

// groupBy에 따라 다른 키로 집계
const key = row[groupBy]; // '소재명' 또는 '파일명'
```

### 4. 테스트 검증 완료
**파일**: `test_auto.html` (자동 테스트 페이지)
- **PHPH.csv 기준 테스트 결과**:
  
| 분석 기준 | 소재 수 | Top 1 | 최고 점수 | 평균 점수 |
|----------|--------|-------|----------|----------|
| **소재명** | 23개 | A-Character-Adventure01A-DA | 98.5점 | 52.2점 |
| **파일명** | 59개 | 251104_BNR_A-Character-Adventure01A-DA_L_1200x628_EN | 94.7점 | 50.8점 |
| **차이** | **36개** | ✅ 다른 소재 | ✅ 다름 | ✅ 다름 |

**해석**:
- ✅ **소재명 기준 (23개)**: 광고 카피가 동일한 여러 크리에이티브를 하나로 통합 집계
  - 예: "A-Character-Adventure01A-DA"라는 소재명으로 묶인 모든 파일의 성과 합산
  - 용도: 광고 메시지/카피의 전반적 성과 평가
  
- ✅ **파일명 기준 (59개)**: 각 크리에이티브 파일을 개별적으로 집계
  - 예: "251104_BNR_A-Character-Adventure01A-DA_L_1200x628_EN.jpg" 고유 파일별 성과
  - 용도: 특정 크리에이티브의 세부 성과 추적 및 소재 피로도 분석

## 📊 영향

### Total Score 재계산
- **분석 기준 변경 시 모든 지표 재집계**:
  - 전환, 비용, 노출수, 클릭수 합계 재계산
  - CPA = 비용 / 전환
  - IPM = (전환 / 노출수) × 1000
  - Rank 점수 (Percentile) 재산출
  - **Total Score = 전환Rank(40%) + CPARank(35%) + IPMRank(25%)**

### 기간 필터와의 연동
- ✅ 기간 필터 + 분석 기준 옵션 **동시 적용**
- 순서:
  1. 날짜 범위 필터링 (2026-04-06 ~ 2026-04-07)
  2. BNR/VID 타입 필터링
  3. 선택된 기준(소재명/파일명)으로 집계
  4. Total Score 산출
