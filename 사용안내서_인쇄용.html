2.1 필수 컬럼 및 자동 인식 컬럼

아래 컬럼명을 포함한 CSV 파일을 준비합니다. 다른 이름도 자동 인식되므로 원본 CSV를 그대로 사용해도 됩니다.

표준 컬럼명자동 인식되는 다른 이름필수설명
소재명Ad Name, 광고명, Creative Name, 파일명필수소재 식별자
유형앱 확장 소재 유형, Type, 소재유형필수BNR / VID (TXT 자동 제외)
전환Conversions, 전환수, conv필수설치/구매 전환수
비용Cost, 비용(원), spend필수광고 집행 비용
노출수Impressions, 노출, impr필수광고 노출 횟수
일Date, 날짜, Campaign Date권장날짜 필터·피로도 기간 설정용
캠페인Campaign, 캠페인명권장캠페인별 필터링용
클릭수Clicks, 클릭선택CTR 계산용
Revenue매출, 수익, revenue선택ROAS 계산용 (없으면 자동 제외)
⚠️ 업로드 주의사항 · 파일 형식: CSV (쉼표 구분)만 지원 — .xlsx 불가
· 인코딩: UTF-8 권장 (한글 깨짐 방지)
· TXT 유형 소재는 자동 제외됩니다 (BNR/VID만 분석)
· 비수치 데이터(N/A, -, 빈칸)는 자동으로 0 처리됩니다

📊 3. Step 1 — 소재 성과 분석
3.1 점수 계산 (4개 지표 가중치)

Total Score는 4개 지표를 Rank 기반으로 계산한 후 가중치를 적용하여 산출합니다.

지표기본 가중치계산 방법의미
전환수25%Rank 기반 — 전환 많을수록 높은 점수실제 성과 (설치/구매)
CPA25%Rank 기반 — CPA 낮을수록 높은 점수비용 효율성
IPM25%Rank 기반 — IPM 높을수록 높은 점수노출 대비 전환력
ROAS25%Rank 기반 — ROAS 높을수록 높은 점수투자 수익률 (Revenue 없으면 자동 제외)
💡 가중치 프리셋 가이드 슬라이더로 직접 조정하거나 아래 추천 프리셋을 참고하세요.

· 균형형 (기본): 전환 25% | CPA 25% | IPM 25% | ROAS 25% — 일반적인 분석
· 전환 극대화: 전환 40% | CPA 30% | IPM 15% | ROAS 15% — UA 캠페인 중점
· 효율 최적화: 전환 20% | CPA 40% | IPM 30% | ROAS 10% — 예산 제약 시
· 수익 극대화: 전환 15% | CPA 15% | IPM 10% | ROAS 60% — LTV 높은 게임
📋 필터 옵션
  • 소재 유형: BNR (이미지) / VID (동영상) — TXT는 분석에서 자동 제외
  • 캠페인: 단일 선택 또는 전체 (캠페인 드롭다운)
  • 날짜 범위: 시작일 ~ 종료일 직접 입력
  • 집계 기준: 소재명 기준 또는 파일명 기준 선택
3.2 등급 기준 및 결과 해석
등급점수 범위색상권장 액션
🏆 최우수≥ 80점진한 녹색예산 즉시 증액 (Scale-up)
🌟 우수60~79점녹색현행 운영 유지, 신규 오디언스 테스트
✅ 양호40~59점파란색크리에이티브 요소 일부 최적화
⚠️ 보통20~39점주황색타겟·입찰 재검토 후 개선
🔴 개선 필요< 20점빨간색즉시 교체 또는 소재 리프레시
✨ AI 인사이트 — 소재 성과 분석 버튼 점수 계산 결과 화면 상단의 "✨ AI 인사이트 생성" 버튼을 누르면
고효율/저효율 소재의 공통 태그 분석 + 포맷 비교 + 즉시 권고를 자동 생성합니다.
(Gemini API 키 설정 필요 → 5번 섹션 참고)

🔍 3.3 피로도 분석 (기간 비교)

두 기간의 소재 성과를 비교하여 순위 변동과 CPA 변화율을 추적합니다. 피로도를 조기 감지하여 교체 시점을 정확히 결정할 수 있습니다.

기간 설정
1
기준 기간 설정 — 비교의 기준이 되는 과거 기간 입력 (예: 3/01~3/31)
2
비교 기간 설정 — 현재 또는 최근 기간 입력 (예: 4/01~4/13)
3
"자동 분할" 버튼 — 전체 데이터 기간을 자동으로 절반씩 나눠 설정
4
"🔍 피로도 분석 실행" 버튼 클릭 → 결과 자동 표시
피로도 상태 기준
🟢
Fresh
CPA ≤ −20%
→ 예산 증액 (Scale-up)
🔵
Stable
−20% ~ 0%
→ 현행 유지 (Maintain)
🟡
Warning
0% ~ +20%
→ 모니터링 강화 (Watch)
🔴
Tired
CPA > +20%
→ 즉시 교체 (Replace)
결과 화면 구성
영역내용
통계 요약 카드Fresh·Stable·Warning·Tired 소재 수, 전체 CPA 변화율 평균
순위 변동 테이블기준기간 순위 → 비교기간 순위 변동 (▲상승 / ▼하락), CPA 변화율, 피로도 뱃지
순위 변동 차트Top 10 소재의 기준 vs 비교 순위 시각화 (Y축 역순 — 위로 갈수록 순위 상승)
✨ AI 인사이트 — 순위 변동 원인 AI 분석 버튼 피로도 분석 결과 화면의 "✨ 순위 변동 원인 AI 분석" 버튼을 누르면
· 급등 소재의 태그 기반 원인 분석 (CPA·전환수 수치 인용)
· 하락 소재의 CPA 증가 트렌드 진단 (피로 집중 소재 명시)
· 교체 우선순위 (CPA 증가율 순) + 운영 액션 자동 생성
⚠️ 피로도 분석 핵심 규칙 · 순위 기준: Total Score 기준 순위 비교 (점수 계산과 일관성)
· 피로도 판정 기준: CPA 변화율 (CPA 악화 = 피로도)
· TXT 소재: 피로도 분석에서도 완전 제외 (BNR/VID만 분석)

📉 5.3 피로도 AI 인사이트 (riseReason)

피로도 분석 결과 화면의 "✨ 순위 변동 원인 AI 분석" 버튼으로 실행합니다.
순위 변동이 큰 소재(급등·급락)를 자동 선별하여 태그 기반 원인을 분석합니다.

분석 데이터 구성 (AI에 전달되는 정보)
데이터 항목내용역할
순위 변동▲N위 / ▼N위 (Total Score 기준)급등·급락 선별 기준
CPA 변화기준기간 CPA → 비교기간 CPA (변화율 %)피로도 판정 핵심 지표
전환수 변화기준 전환 → 비교 전환성과 실제 변화 확인
피로도 상태fresh / stable / warning / tired피로 수준 컨텍스트
meta 태그hooking·emotion·mechanic 등 2~3개태그 기반 원인 분석 핵심
★피로집중 목록CPA +20% 이상 소재 자동 추출교체 우선순위 자동 결정
출력 슬롯 구성
📈 급등 소재 원인 무료·유료
· 급등 소재명 + CPA 수치 → 공통 태그([태그명]) 기반 원인 2줄
· CPA 증가 트렌드 요약 (★피로집중 소재 명시)
· 교체 우선순위 (CPA 증가율 순)
· 급등 소재 태그를 교체 소재에 적용하는 방향
📉 피로도 진단 유료
하락 소재별 CPA 변화율 + 피로 패턴·취약 태그 분석 2줄

🔄 운영 액션 유료
ON 확대·OFF 실행·신소재 방향 3줄 (태그 기반 제작 가이드 포함)
✅ 실측 검증 결과 (무료 플랜 예시) finishReason: STOP (잘림 없음) ✅
입력: ~824 토큰 / 출력: ~594 토큰 / 합계: ~1,418 토큰 (무료 한도 8,192 대비 17.3%)
출력 예시: "A-Character-Keyart01A-DA CPA 6,638→762(-88.5%) — '키아트/수집욕/감동' 태그 조합이 신규 유저의 강력한 수집 욕구와 감성적 연결을 자극하여 전환 3→21건으로 급증했습니다."

❓ 6. 자주 묻는 질문 (FAQ)
"컬럼을 찾을 수 없습니다" 오류가 나와요.
✅ 해결 방법 1. CSV를 엑셀로 열어 첫 번째 행(헤더) 확인
2. 컬럼명을 표준명으로 수정 (예: "Date" → "일", "Conversions" → "전환")
3. UTF-8 CSV 형식으로 저장 후 재업로드
4. "다른 이름으로 저장 → CSV UTF-8(쉼표로 분리)"로 저장
지표가 모두 0으로 나와요.
✅ 해결 방법 1. 전환·비용·노출수 컬럼의 값이 숫자인지 확인
2. 쉼표 포함 숫자("1,234") → 숫자만("1234")으로 변경
3. "N/A", "-", 빈칸은 시스템이 자동으로 0으로 처리 (정상 동작)
ROAS 점수가 0이에요.
Revenue 컬럼이 없거나 모든 값이 0인 경우 ROAS를 자동 제외합니다. ROAS 가중치는 나머지 3개 지표에 자동 재배분됩니다. (예: 25%+25%+25%+25% → 33.3%+33.3%+33.3%+0%)
TXT 소재가 분석에서 빠졌어요.
정상입니다. TXT 유형(광고 제목/설명 텍스트)은 크리에이티브 태그 정보가 없어 분석 대상에서 자동 제외됩니다. BNR·VID만 분석됩니다.
Gemini AI 인사이트 버튼이 비활성화되어 있어요.
API 키 미설정 상태입니다. 네비게이션 상단 "✨ AI 설정" 버튼을 클릭하여 Google AI Studio에서 발급한 API 키를 입력하세요.
AI 인사이트가 429 오류로 실패해요.
무료 API는 분당 15회 한도입니다. 1분 후 재시도하거나 유료 플랜(분당 1,000회)으로 전환하세요.
Step 2 군집화를 하려면 CSV에 태그 컬럼이 반드시 있어야 하나요?
네, 필수입니다. hooking, emotion, mechanic 등 태그 컬럼이 없으면 군집화가 실행되지 않습니다. CSV 생성 시 소재별 태그를 사전에 입력해두세요.
피로도 분석에서 "없음"으로 표시되는 소재가 있어요.
기준 기간에만 있거나 비교 기간에만 있는 소재는 "신규" 또는 "사라짐"으로 별도 표시됩니다. 두 기간 모두 존재하는 소재만 순위 변동이 계산됩니다.

📋 7. 빠른 참조 (Cheat Sheet)
📊 가중치 프리셋
  • 균형형 (기본): 전환 25% | CPA 25% | IPM 25% | ROAS 25%
  • 전환 극대화: 전환 40% | CPA 30% | IPM 15% | ROAS 15%
  • 효율 최적화: 전환 20% | CPA 40% | IPM 30% | ROAS 10%
  • 수익 극대화: 전환 15% | CPA 15% | IPM 10% | ROAS 60%
🏆 소재 등급 기준
  • 🏆 최우수 ≥ 80점 → 즉시 Scale-up
  • 🌟 우수 60~79점 → 현행 유지
  • ✅ 양호 40~59점 → 일부 최적화
  • ⚠️ 보통 20~39점 → 타겟 재검토
  • 🔴 개선 필요 <20점 → 즉시 교체
📉 피로도 상태 기준
  • 🟢 Fresh: CPA ≤ −20% → Scale-up
  • 🔵 Stable: −20% ~ 0% → 유지
  • 🟡 Warning: 0% ~ +20% → 모니터링
  • 🔴 Tired: CPA > +20% → 즉시 교체
🔬 군집 등급 기준
  • S 최우수 ≥ 75점 → 예산 집중 투입
  • A 우수 60~74점 → 유사 소재 추가 제작
  • B 양호 45~59점 → 태그 보강 (Pivot)
  • C/D 보통·개선 필요 → Exit 검토
✨ AI 인사이트 플랜 비교
  • 🆓 무료: 1회 호출 · 1슬롯 · 분당 15회
  • 💎 유료: 2회 호출 · 전체 슬롯 · 분당 1,000회
  • 429 오류 → 1분 후 재시도
  • 403 오류 → API 키 재확인
📂 주간 분석 루틴 (권장)
  • ① 전주 CSV 준비 (UTF-8 저장)
  • ② Step 1 업로드 → 점수 계산
  • ③ 피로도 분석 (전전주 vs 전주)
  • ④ AI 인사이트 생성 → 교체 소재 결정
