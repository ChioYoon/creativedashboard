## 🐛 발견된 문제

### 증상
ROAS 값이 동일한 소재들이 서로 다른 ROAS 점수를 받는 현상

### 예시
```
기존 (v1.5, 잘못됨):
- 소재A: ROAS 2.0 → Rank 1 → 점수 100
- 소재B: ROAS 2.0 → Rank 2 → 점수 75  ❌ 동일한 ROAS인데 다른 점수!
- 소재C: ROAS 2.0 → Rank 3 → 점수 50  ❌ 동일한 ROAS인데 다른 점수!
- 소재D: ROAS 1.5 → Rank 4 → 점수 25
```

### 원인
정렬 후 단순히 인덱스 순서대로 Rank를 부여하여, 동일한 값을 가진 항목들이 다른 Rank를 받음

---

## ✅ 해결 방법

### 동점자 처리 로직 추가
```javascript
function assignRankWithTies(sortedArray, valueGetter) {
  let currentRank = 1;
  let previousValue = null;
  
  sortedArray.forEach((item, index) => {
    const currentValue = valueGetter(item);
    
    // 이전 값과 동일하면 (오차 0.0001 이내)
    if (previousValue !== null && Math.abs(currentValue - previousValue) < 0.0001) {
      // 동점: 이전과 같은 Rank 유지
    } else {
      // 새로운 값: Rank 업데이트 (현재 인덱스 + 1)
      currentRank = index + 1;
    }
    
    item.assignedRank = currentRank;
    previousValue = currentValue;
  });
}
```

### 개선 결과
```
신규 (v1.5.1, 올바름):
- 소재A: ROAS 2.0 → Rank 1 → 점수 100  ✅
- 소재B: ROAS 2.0 → Rank 1 → 점수 100  ✅ 동일 Rank, 동일 점수
- 소재C: ROAS 2.0 → Rank 1 → 점수 100  ✅ 동일 Rank, 동일 점수
- 소재D: ROAS 1.5 → Rank 4 → 점수 50   ✅ 3명 동점 후 4위로 건너뜀
```

### 1. 공정성
- ✅ 동일한 성과는 동일한 평가를 받아야 함

- ✅ 미세한 소수점 차이 무시 (오차 < 0.0001)

### 4. 의사결정

- ✅ 다른 지표(전환수, CPA, IPM)로 차별화

Math.abs(currentValue - previousValue) < 0.0001

### 순위 건너뛰기
```
예: 5개 소재 중 3개 동점 1위
- 1위: 3개 (점수 100)
- 4위: 1개 (점수 50)  ← 2위, 3위 건너뜀
- 5위: 1개 (점수 25)

점수 계산:
- Rank 1: (5-1+1)/5 × 100 = 100
- Rank 4: (5-4+1)/5 × 100 = 40
- Rank 5: (5-5+1)/5 × 100 = 20
```
