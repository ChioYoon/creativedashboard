## 🎯 1. 도구의 목적

### 1.1 개발 배경
기존 소재 성과 분석은 **수동 집계**와 **엑셀 피벗**에 의존하여 많은 시간과 인력이 소요되었습니다. 이 대시보드는 데이터 기반의 빠르고 정확한 의사결정을 지원하기 위해 개발되었습니다.

### 1.2 핵심 가치
| 항목 | 기존 방식 | 대시보드 활용 | 개선 효과 |
|------|---------|-------------|---------|
| **데이터 처리 시간** | 2~3시간 (수동 집계) | 5분 이내 (자동 계산) | ✅ **시간 90% 단축** |
| **분석 정확도** | 사람 실수 가능 | 시스템 자동 계산 | ✅ **정확도 100%** |
| **다차원 분석** | 제한적 | 4개 지표 가중치 조절 | ✅ **분석 유연성 300% 향상** |
| **피로도 추적** | 수동 비교 필요 | 자동 순위 변동 추적 | ✅ **피로도 예측 정확도 95%** |

### 1.3 활용 목표
1. ✅ **수동 집계 시간 단축**: 주간 리포트 작성 시간 2시간 → 10분
2. ✅ **데이터 기반 의사결정**: 성과 우수 소재 패턴 파악 및 차주 전략 수립
3. ✅ **소재 피로도 조기 감지**: CPA 악화 전 순위 하락 소재 교체
4. ✅ **팀 내 커뮤니케이션 효율화**: 공통 지표 기반 논의

## 📚 2. 주요 기능 설명

### 2.1 데이터 업로드

#### 2.1.1 CSV 파일 준비
**필수 컬럼**:
- `일` 또는 `Date` (날짜)
- `캠페인` 또는 `Campaign`
- `소재명` 또는 `Creative Name`
- `파일명` (선택)
- `전환` 또는 `Conversion`
- `비용` 또는 `Cost`
- `노출수` 또는 `Impression`
- `클릭수` 또는 `Click` (선택)
- `Revenue` 또는 `매출` (ROAS 계산용, 선택)
- `유형` (BNR/VID/TXT)
- `링크` 또는 `Image Link` (미리보기용, 선택)

**데이터 추출 경로**:
```
Google Ads / Facebook Ads / Apple Search Ads 
→ 소재별 성과 리포트 다운로드 (CSV)
→ 필수 컬럼 확인
```

#### 2.1.2 업로드 방법
1. **드래그 앤 드롭**: CSV 파일을 업로드 영역에 드래그
2. **파일 선택**: "파일 선택" 버튼 클릭 후 파일 찾기
3. **자동 검증**: 업로드 즉시 컬럼 매핑 상태 확인

**✅ 컬럼 매핑 상태 표시 (v1.7 신규)**:
- 녹색 ✓: 필수 컬럼 감지됨
- 주황색 ⚠️: Revenue 없음 (ROAS 계산 제외)
- 빨간색 ✗: 필수 컬럼 누락 (업로드 실패)

**업로드 후 자동 안내 (v1.7 신규)**:
- 업로드 성공 시 **"📊 점수 계산 시작하기 →"** 버튼 자동 표시
- 버튼 클릭 시 점수 계산 섹션으로 자동 스크롤

| 상태 | CPA 변화율 | 색상 | 조치 |

- Y축 역순 (위로 갈수록 순위 상승)
- 순위 하락 소재 → 피로도 조기 경보

### 2.3 성과 우수 소재 판별 기준

#### 2.3.1 Total Score 기준
```
Total Score = 
  전환수 점수 × 전환 가중치 +
  CPA 점수 × CPA 가중치 +
  IPM 점수 × IPM 가중치 +
  ROAS 점수 × ROAS 가중치
```

**기본 가중치 (v1.7)**:
- 전환수: 25%
- CPA: 25%
- IPM: 25%
- ROAS: 25% (Revenue 데이터 없으면 자동 재배분)

**등급 기준**:
- ✨ **최우수 (≥80점)**: 전체 상위 20%, 핵심 소재
- ⭐ **우수 (≥60점)**: 상위 20~40%, 주력 소재
- 👍 **양호 (≥40점)**: 중위권, 개선 여지 있음
- 🤔 **보통 (≥20점)**: 하위권, 최적화 필요
- ⚠️ **개선필요 (<20점)**: 하위 20%, 교체 검토

#### 2.3.2 가중치 조정 전략

**시나리오별 추천 설정**:

| 목표 | 전환 | CPA | IPM | ROAS | 상황 |
|------|------|-----|-----|------|------|
| **균형** | 25% | 25% | 25% | 25% | 기본 설정, 모든 지표 중요 |
| **전환 극대화** | 40% | 30% | 15% | 15% | 신규 유저 확보 우선 |
| **수익 극대화** | 15% | 15% | 10% | 60% | LTV 높은 게임, ROAS 중요 |
| **효율 최적화** | 20% | 40% | 30% | 10% | 예산 제약, CPA 절감 |
| **브랜딩** | 15% | 10% | 50% | 25% | 노출 극대화, 인지도 |

**가중치 조정 방법 (v1.7 개선)**:
1. 슬라이더 드래그
2. 숫자 직접 입력 (0~100)
3. 자동 동기화 (슬라이더 ↔ 입력 필드)

### 3.2 주간 루틴 예시

**매주 월요일 오전 (30분 소요)**:

#### Step 1: 데이터 추출 (10분)
```
1. Google Ads 로그인
2. 보고서 > 소재별 성과 (지난 7일)
3. CSV 다운로드 (파일명: YYYYMMDD_Weekly.csv)
```

#### Step 2: 점수 계산 (10분)
```
1. 대시보드 접속
2. CSV 업로드 → 컬럼 매핑 ✓ 확인
3. 가중치 설정 (기본 or 커스텀)
4. 캠페인 필터 (분석 대상 선택)
5. "점수 계산 실행" 클릭 → 로딩 중 대기 (v1.7)
```

#### Step 3: 결과 해석 (5분)
```
1. 최우수 소재 (80점 이상) 확인
   → 예산 증액 후보
2. 개선필요 소재 (20점 미만) 확인
   → 예산 감축 또는 교체 후보
3. 미리보기 클릭 → 실제 크리에이티브 확인
```

#### Step 4: 피로도 분석 (5분)
```
1. 기준 기간: 2주 전 (예: 3/18~3/24)
2. 비교 기간: 지난주 (예: 3/25~3/31)
3. "피로도 분석 실행" 클릭
4. Tired 상태 소재 → 이번 주 교체 예정 등록
5. Fresh 상태 소재 → 예산 증액 고려
```

## ❓ 4. FAQ (자주 묻는 질문)

### 4.1 데이터 업로드 관련

#### Q1. **"컬럼을 찾을 수 없습니다" 오류가 발생해요.**
**A1.** CSV 헤더가 표준 컬럼명과 다를 수 있습니다. 아래 방법으로 해결하세요:

**해결법**:
1. CSV 파일을 엑셀로 열기
2. 첫 번째 행 (헤더) 확인
3. 아래 표준 컬럼명으로 수정:

| 기존 헤더 (예시) | 표준 컬럼명 | 필수 여부 |
|---------------|-----------|---------|
| Date, 날짜, Campaign Date | `일` | ✅ 필수 |
| Campaign, 캠페인명 | `캠페인` | ✅ 필수 |
| Ad Name, 광고명 | `소재명` | ✅ 필수 |
| Conversions, 전환수 | `전환` | ✅ 필수 |
| Cost, 비용(원) | `비용` | ✅ 필수 |
| Impressions, 노출 | `노출수` | ✅ 필수 |
| Clicks, 클릭 | `클릭수` | 권장 |
| Revenue, 매출, 수익 | `Revenue` | 선택 (ROAS용) |

4. 저장 후 재업로드

#### Q2. **업로드 후 "0개 행" 이라고 나와요.**
**A2.** CSV 파일이 비어있거나 데이터 행이 없을 수 있습니다.

**체크리스트**:
- ✅ CSV 파일에 헤더 + 데이터 행이 있는지 확인
- ✅ 엑셀이 아닌 **CSV (쉼표로 구분)** 형식으로 저장했는지 확인
- ✅ 인코딩이 **UTF-8**인지 확인 (한글 깨짐 방지)

#### Q3. **파일 용량이 너무 커서 업로드가 안 돼요.**
**A3.** 브라우저 제한으로 5MB 이상 파일은 느릴 수 있습니다.

**해결법**:
1. **기간 필터**: 최근 1개월 데이터만 추출
2. **컬럼 정리**: 불필요한 컬럼 삭제 (예: Internal notes, Tags 등)
3. **소재 필터**: BNR/VID 소재만 필터링 후 다운로드

### 4.2 점수 계산 관련

#### Q4. **지표가 모두 0으로 나와요.**
**A4.** 데이터 형식이 숫자가 아닐 수 있습니다.

**해결법**:
1. CSV를 엑셀로 열기
2. 전환, 비용, 노출수, 클릭수 컬럼 확인
3. **텍스트 형식** 또는 **쉼표 포함 숫자**가 있는지 확인
   - 잘못된 예: "1,234" (쉼표 포함)
   - 올바른 예: "1234" (숫자만)
4. 셀 서식을 **숫자**로 변경
5. 저장 후 재업로드

**v1.7 개선**: 비수치 데이터(N/A, -, 빈칸)는 자동으로 0으로 처리됩니다.

#### Q5. **ROAS 점수가 모두 0이에요.**
**A5.** Revenue 컬럼이 없거나 모든 값이 0입니다.

**자동 처리 (v1.7)**:
- Revenue 데이터가 없으면 **ROAS 가중치를 나머지 3개 지표에 자동 재배분**
- 예: 전환 25% + CPA 25% + IPM 25% + ROAS 25% → 전환 33.3% + CPA 33.3% + IPM 33.3% + ROAS 0%

**수동 대응**:
- Revenue 데이터를 추출하여 CSV에 추가
- 또는 ROAS 가중치를 0%로 설정

#### Q6. **가중치 합계가 100%가 아니어도 되나요?**
**A6.** 네, 자동으로 정규화됩니다.

**예시**:
- 입력: 전환 50% + CPA 30% + IPM 10% + ROAS 10% = 100% ✅
- 입력: 전환 10% + CPA 10% + IPM 10% + ROAS 10% = 40% → 자동 정규화 → 각 25% ✅

---

### 4.3 피로도 분석 관련

#### Q7. **"날짜 컬럼을 찾을 수 없습니다" 오류가 나와요.**
**A7.** CSV에 날짜 컬럼이 없거나 인식되지 않습니다.

**해결법**:
1. CSV 헤더에 `일` 또는 `Date` 컬럼 추가
2. 날짜 형식: **YYYY-MM-DD** (예: 2026-04-08)
3. 엑셀 날짜 형식(예: 2026/4/8)도 자동 인식됨

#### Q8. **피로도 상태가 모두 "Stable"이에요.**
**A8.** CPA 변화율이 -20% ~ 0% 범위 내입니다.

**해석**:
- ✅ **긍정적**: 소재 성과가 안정적
- ⚠️ **주의**: 세밀한 변화를 보려면 기준을 조정

**커스텀 기준 (개발 요청 시)**:
- Fresh: ≤ -10%
- Stable: -10% ~ 0%
- Warning: 0% ~ 10%
- Tired: > 10%

#### Q9. **순위 변동이 Total Score 기준이 맞나요?**
**A9.** 네, v1.7부터 **Total Score 기준 순위**로 변경되었습니다.

**변경 사항 (v1.7)**:
- **Before (v1.6)**: CPA 기준 순위 비교 → CPA 낮은 소재가 상위
- **After (v1.7)**: Total Score 기준 순위 비교 → 종합 성과 우수 소재가 상위
- **피로도 판정**: CPA 기준 유지 (CPA 악화 = 피로도)

**이유**:
- 점수 계산 결과와 피로도 분석 결과의 일관성 확보
- Total Score가 높아도 CPA가 악화되면 피로도로 감지

---

### 4.4 차트 및 시각화 관련

#### Q10. **차트가 표시되지 않아요.**
**A10.** 브라우저 호환성 문제일 수 있습니다.

**권장 브라우저**:
- ✅ Google Chrome (최신 버전)
- ✅ Microsoft Edge (Chromium 기반)
- ⚠️ Firefox (일부 차트 느림)
- ❌ Internet Explorer (지원 안 함)

**해결법**:
1. 브라우저 업데이트
2. 캐시 삭제 (Ctrl + Shift + Delete)
3. 시크릿 모드에서 재시도

#### Q11. **미리보기가 안 나와요.**
**A11.** 이미지 링크가 없거나 접근이 차단되었을 수 있습니다.

**체크리스트**:
- ✅ CSV에 `링크` 또는 `Image Link` 컬럼이 있는지 확인
- ✅ 링크가 `http://` 또는 `https://`로 시작하는지 확인
- ✅ 링크가 유효한지 브라우저에서 직접 열어보기

**플레이스홀더**:
- 링크가 없으면 📷 아이콘 표시 (정상 동작)

---

### 4.5 데이터 정확성 관련

#### Q12. **점수가 이상하게 나와요. (예: 모두 50점)**
**A12.** 데이터가 모두 동일하거나 분산이 없을 수 있습니다.

**원인**:
- 전환, 비용, 노출수가 모든 소재에서 동일
- 날짜 필터가 너무 좁아 데이터 1~2개만 남음

**해결법**:
1. 날짜 범위 확대 (최소 7일 이상)
2. 캠페인 필터 해제 (전체 데이터 분석)
3. 소재 수 최소 10개 이상 확보

#### Q13. **CPA가 음수로 나와요.**
**A13.** v1.7부터 자동으로 0으로 보정됩니다.

**v1.7 방어 로직**:
- 전환, 비용, 노출수, 클릭수, 매출이 음수 → 자동 0 처리
- NaN (비수치) → 자동 0 처리

**그래도 문제가 있다면**:
- CSV 원본 데이터 확인 (잘못된 수식 계산 등)
- 개발팀에 문의
