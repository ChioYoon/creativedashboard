## 📊 1. 데이터 정합성 검증 결과

### ✅ KPI 계산식 정합성
| 지표 | 현재 구현 | 마케팅 표준 | 검증 결과 |
|------|---------|-----------|---------|
| **CPA** | 비용 / 전환 | Cost / Conversion | ✅ 일치 |
| **IPM** | (전환 / 노출수) × 1000 | (Conv / Imp) × 1000 | ✅ 일치 |
| **ROAS** | 매출 / 비용 | Revenue / Cost | ✅ 일치 |
| **전환율** | (전환 / 노출수) × 100 | (Conv / Imp) × 100 | ✅ 일치 |
| **CTR** | (클릭수 / 노출수) × 100 | (Click / Imp) × 100 | ✅ 일치 |

### 🔴 발견된 문제점
1. **CPI 지표 미구현**: 설치 수 데이터 필요 (현재 전환 데이터로 대체 사용 중)
2. **Revenue=0 처리 부족**: PHPH_upload.csv 전체 행이 Revenue=0 → ROAS 점수 무의미
3. **TXT 유형 제외**: 광고 제목/설명 텍스트 성과 추적 불가

### ✅ v1.6 개선 완료
- Revenue=0 → **ROAS 가중치 자동 재배분**
- TXT 유형 → **필터 옵션 추가**

## 🛡️ 2. 예외 처리 로직 분석

### Before (v1.5.1)
| 예외 상황 | 처리 방식 | 위험도 |
|---------|---------|--------|
| 비수치 텍스트 ("N/A") | ❌ 미처리 | 🔴 높음 (NaN 발생) |
| 음수 값 | ❌ 미처리 | 🟡 중간 (순위 왜곡) |
| Revenue=0 | ❌ 미처리 | 🟡 중간 (ROAS 무의미) |

### After (v1.6)
| 예외 상황 | 처리 방식 | 효과 |
|---------|---------|------|
| 비수치 텍스트 | ✅ `safeParseNumber()` | NaN 100% 차단 |
| 음수 값 | ✅ `Math.max(0, value)` | 데이터 무결성 확보 |
| Revenue=0 | ✅ 가중치 자동 재배분 | 안정적 점수 계산 |

### 개선 코드
```javascript
// v1.6 추가: 안전한 숫자 파싱
function safeParseNumber(value, defaultValue = 0) {
  if (value === undefined || value === null || value === '') return defaultValue;
  const parsed = parseFloat(value);
  return isNaN(parsed) ? defaultValue : Math.max(0, parsed);
}

// 집계 시 적용
groups[key].전환 += safeParseNumber(row['전환']);
groups[key].비용 += safeParseNumber(row['비용']);

// 음수 보정
g.전환 = Math.max(0, g.전환);
g.비용 = Math.max(0, g.비용);

// Revenue=0 처리
const hasRevenue = creatives.some(c => c.매출 > 0);
if (!hasRevenue && roasWeight > 0) {
  // ROAS 가중치 재배분
}
```
