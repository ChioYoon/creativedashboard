# Pepp Heroes 소재 분석 시스템 (대시보드 + 파이프라인)

## 프로젝트 개요
Pepp Heroes 모바일 게임의 광고 소재(BNR/VID) 분석을 위한 **2-Page 시스템**입니다.
- `pipeline.html` — **분석 파이프라인**: CSV 업로드 → 검증 → 자동 군집화 → 차수 비교 → 인사이트 확정 (5단계)
- `index.html` — **성과 대시보드**: 군집 인사이트, 소재 비교표, 미리보기, 태그 영향도 시각화

## 파이프라인 시스템 (pipeline.html) – 4단계

### STEP 1: 데이터 업로드
- 차수명·담당자·메모 입력
- CSV 드래그앤드롭 / 파일 선택 / 샘플 데이터 테스트
- 파일 파싱 후 BNR+VID만 분석 대상으로 자동 분리
- 업로드 전 6개 항목 체크리스트
- **자동 컬럼 정규화**: 실제 UA CSV 헤더를 파이프라인 표준 컬럼명으로 자동 매핑
- **마케터 인사이트 자동 탐지**: `marketer_insight` 컬럼 자동 인식 및 키워드 추출

### STEP 2: 데이터 품질 검증
- 필수 컬럼 누락 자동 감지
- 중복 소재명, 음수값, 전환 0 건 이상치 탐지
- 태그 품질 (평균 태그 수, 누락 비율) 측정
- **태그 컬럼 자동 탐지**: CSV 헤더에서 태그 컬럼과 인사이트 컬럼 자동 분리
- **인사이트 키워드 고도화**: marketer_insight 필드에서 의미 있는 키워드만 추출하여 [MI] 태그로 변환
- 검증 결과 통과/주의/실패 등급 표시

### STEP 3: 자동 군집화 분석
- **Total Score 산출**: 전환수(25%) · CPA 역가중(25%) · IPM(25%) · ROAS(25%) – 슬라이더로 조정 가능
  - **전환수 점수**: Rank 기반 (n - Rank + 1) / n × 100 (v1.5 변경)
  - **CPA 점수**: Rank 기반 (낮을수록 높은 점수)
  - **IPM 점수**: Rank 기반 (높을수록 높은 점수)
  - **ROAS 점수**: Rank 기반 (높을수록 높은 점수, v1.5 신규)
- **태그 동시출현 기반 클러스터링**: Union-Find 알고리즘, 코어 태그 지지율·동시출현 기준 설정 가능
- **마케터 인사이트 태그 우대**: [MI] 태그는 군집화 시 1.5배 가중치 적용 및 노이즈 필터링 면제
- **이전 차수 군집과 유사성 비교** → 기존 군집명 유지 또는 NEW 배지 부여
- 군집별 TOP3 소재, 성과 비교 차트, 소재-군집 매핑 테이블

### STEP 4: 인사이트 확정 & 저장
- 최종 KPI 요약 (소재 수, 군집 수, 평균/최고 Score, 최우수 소재 수)
- 승리 공식·중단 패턴 텍스트 입력 (이전 차수 값 자동 노출)
- 전체 차수 히스토리 테이블
- **저장 시 localStorage에 누적** → 다음 차수 군집화에 자동 반영

## 🚀 빠른 시작 가이드 (`step1_integrated.html`)

### Step 1: 페이지 열기
브라우저에서 `step1_integrated.html` 파일을 엽니다.

### Step 2: CSV 업로드
1. **드래그 앤 드롭**: CSV 파일을 화면 상단 업로드 영역에 드래그
2. **파일 선택**: 업로드 영역을 클릭하여 파일 브라우저에서 선택
3. ✅ 업로드 완료 메시지와 데이터 요약 확인

### Step 3: 점수 계산
1. 📊 상단 네비게이션에서 **"점수 계산"** 버튼 클릭
2. (선택) 가중치 슬라이더 조정
3. (선택) 분석 기준(소재명/파일명), 캠페인, 날짜 범위 설정
4. **"📊 점수 계산 실행"** 버튼 클릭
5. 📋 분석 결과 섹션으로 자동 이동하여 결과 확인

### Step 4: 피로도 분석
1. 🔍 상단 네비게이션에서 **"피로도 분석"** 버튼 클릭
2. (선택) 기준/비교 기간 조정 (기본값: 자동 분할)
3. **"🔍 피로도 분석 실행"** 버튼 클릭
4. 📈 통계 섹션으로 자동 이동하여 결과 확인
5. 📉 차트 섹션에서 Top 10 순위 변동 시각화 확인

### Step 5: 결과 해석
- 🟢 **신선** (CPA ≤ -20%) → 예산 증액 (Scale-up)
- 🔵 **안정** (-20% < CPA ≤ 0%) → 현상 유지 (Maintain)
- 🟡 **경고** (0% < CPA ≤ 20%) → 모니터링 강화 (Watch)
- 🔴 **피로** (CPA > 20%) → 교체/리프레시 (Replace)

### 📌 문제 해결
**Q: CSV 파일이 업로드되지 않아요!**
- A1: **✅ v1.2에서 수정됨!** "Cannot read properties of undefined (reading 'length')" 오류 해결
- A2: 브라우저 콘솔(F12)을 열어 오류 메시지 확인
- A3: `js/pipeline-engine.js`와 `js/analysis-engine-v2.js` 파일이 같은 폴더에 있는지 확인
- A4: CSV 파일이 UTF-8 인코딩인지 확인
- A5: 파일 크기가 너무 크지 않은지 확인 (권장: 5MB 이하)
- A6: `test_step1_csv_upload.html`을 열어 자동 테스트 실행 (모든 단계가 ✅이어야 함)

**Q: 날짜 범위가 자동 설정되지 않아요!**
- A: CSV에 "일", "날짜", "일자", "date", "day" 등의 날짜 컬럼이 있는지 확인

**Q: 캠페인 필터가 비어있어요!**
- A: CSV에 "캠페인", "campaign" 등의 캠페인 컬럼이 있는지 확인

**중요**: CPA(Cost Per Action) 변화율을 기준으로 피로도를 판정합니다.

4단계 자동 분류 시스템:
- **🟢 신선 (Fresh)**: CPA 20% 이상 하락 → **성과 크게 개선** → Scale-up 권장
- **🔵 안정 (Stable)**: CPA 0~10% 범위 → **안정적 성과** → 현 상태 유지
- **🟡 주의 (Warning)**: CPA 0~20% 상승 → **경미한 악화** → 모니터링 강화
- **🔴 피로 (Tired)**: CPA 20% 이상 상승 → **성과 크게 악화** → 교체 또는 리프레시 필요

**판정 로직**:
```javascript
// CPA 변화율 = ((비교 기간 CPA - 기준 기간 CPA) / 기준 기간 CPA) × 100
// 음수 = CPA 하락 (좋음), 양수 = CPA 상승 (나쁨)

if (cpaChangeRate <= -20) → 신선 (Fresh)
else if (cpaChangeRate > -20 && cpaChangeRate <= 0) → 안정 (Stable)
else if (cpaChangeRate > 0 && cpaChangeRate < 20) → 주의 (Warning)
else → 피로 (Tired)
```

### ✨ Step 1: 캠페인 필터 추가 + YouTube 임베드 오류 수정 완료

**배경**:

**개선 내용**:

#### 1. 캠페인 필터 기능 추가
- **자동 캠페인 목록 추출**: CSV 업로드 시 '캠페인' 컬럼에서 고유 값 자동 추출
- **드롭다운 선택**: "전체 캠페인" 또는 특정 캠페인 선택
- **동적 재분석**: 캠페인 변경 시 자동으로 필터 적용 및 Total Score 재계산
- **UI 위치**: 분석 설정 섹션, 분석 기준과 기간 필터 사이
- **초기화**: "초기화" 버튼 클릭 시 캠페인 필터도 함께 리셋

**사용 예시**:
- PHPH.csv 업로드 → 1개 캠페인 자동 감지
- 캠페인 선택 → 해당 캠페인 데이터만 집계
- 전환/CPA/IPM Rank 재산출 → Total Score 업데이트

#### 2. YouTube 임베드 오류 153 해결
**문제 원인**: 
- YouTube가 특정 도메인에서 임베드를 제한하는 경우 발생
- 일부 브라우저/환경에서 `youtube.com` 임베드가 차단됨

**해결 방법**:
- ✅ **`youtube-nocookie.com` 사용**: 개인정보 보호 강화 도메인으로 변경
- ✅ **추가 파라미터**: `rel=0` (관련 동영상 최소화), `modestbranding=1` (YouTube 로고 최소화)
- ✅ **권한 확대**: `web-share` 추가
- ✅ **Referrer Policy**: `strict-origin-when-cross-origin` 설정

**변경 전**:
```javascript
iframe.src = `https://www.youtube.com/embed/${videoId}?autoplay=1`;
```

**변경 후**:
```javascript
iframe.src = `https://www.youtube-nocookie.com/embed/${videoId}?autoplay=1&rel=0&modestbranding=1`;
iframe.allow = 'accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share';
iframe.setAttribute('referrerpolicy', 'strict-origin-when-cross-origin');
```

**효과**:
- ✅ 오류 153 해결 (임베드 제한 우회)
- ✅ 사용자 프라이버시 강화 (쿠키 없음)
- ✅ 크로스 브라우저 호환성 개선
- ✅ 안정적인 동영상 재생

### ✨ Step 1: 분석 기준 옵션 추가 완료 (이전 업데이트)

**배경**: 사용자가 첨부한 PHPH.csv 파일에 '소재명'(T열)과 '파일명'(S열)이 별도로 존재하지만, 분석 시 하나의 기준으로만 집계되어 다양한 관점의 분석이 불가능했던 문제

**개선사항**:

1. **소재명/파일명 분석 기준 선택 옵션 추가**
   - **소재명 기준**: 광고 카피(Ad Copy) 단위로 집계 
     - 예: "A-Character-Adventure01A-DA"로 묶인 모든 크리에이티브를 하나로 합산
     - 결과: 23개 소재 (동일 카피를 사용하는 다양한 크리에이티브가 통합됨)
   - **파일명 기준**: 크리에이티브 파일 단위로 집계
     - 예: "251104_BNR_A-Character-Adventure01A-DA_L_1200x628_EN.jpg"처럼 고유한 파일별로 분석
     - 결과: 59개 소재 (각 크리에이티브 파일이 독립적으로 분석됨)

2. **CSV 컬럼 매핑 강화**
   - `plNormalizeColumns()` 함수 개선: '소재명'과 '파일명' 컬럼을 모두 보존
   - 기존 문제: BNR/VID 행에서 '파일명' 값이 '소재명'을 덮어쓰는 문제 해결
   - 새로운 로직: 두 컬럼을 독립적으로 유지하여 사용자가 선택 가능

3. **집계 로직 개선**
   - `aggregateCreativeData()` 함수: `groupBy` 파라미터 지원
   - TXT 유형 자동 제외 (BNR/VID만 분석)
   - 선택된 기준에 따라 다른 집계 결과 생성:
     - 전환, 비용, 노출수, 클릭수 합계
     - CPA, IPM, CTR 재계산
     - 태그 및 인사이트는 첫 번째 행 기준으로 유지

4. **UI/UX 개선**
   - 기간 필터 섹션에 분석 기준 라디오 버튼 추가
   - 기준 변경 시 자동 재분석 실행
   - 결과 테이블에 집계 기준 값 표시 (굵은 글씨)
   - 보조 정보(다른 기준 값)는 작은 글씨로 표시

5. **검증 테스트 완료**
   - `test_auto.html`: 자동 테스트 페이지로 두 기준 비교 검증
   - 실제 PHPH.csv 데이터로 테스트:
     - 소재명 기준: 23개 소재 (최고 점수 98.5점, 777전환)
     - 파일명 기준: 59개 소재 (최고 점수 94.7점, 758전환)
     - 차이: 36개 (정상 작동 확인)

**사용 시나리오**:
- **소재명 기준**: 광고 카피의 전반적인 성과를 평가하고 싶을 때
- **파일명 기준**: 특정 크리에이티브 파일의 세부 성과를 추적하고 피로도를 분석하고 싶을 때

**영향**:
- Total Score가 분석 기준에 따라 다르게 계산됨
- 군집화 분석 시 더 세밀하거나 통합적인 관점 선택 가능
- 소재 피로도 분석 시 파일명 기준으로 일자별 순위 변동 추적 가능

### ✨ marketer_insight 키워드 추출 고도화 완료 (이전 업데이트)

**배경**: 기존 인사이트 키워드 추출 로직에서 "높은", "강조", "마케터 의도" 등 분석 용어와 유의미하지 않은 키워드가 추출되어 분석이 왜곡되는 문제 발생

**개선사항**:

1. **대폭 확장된 불용어 사전 (150+ 단어)**
   - 마케팅/분석 메타 용어: "소재", "분석", "결과", "확인", "전략", "방향", "개선", "최적화" 등
   - 일반 형용사/부사: "높은", "낮은", "강한", "많은", "빠른" 등
   - 분석 프레임 키워드: "마케터", "의도", "타겟", "심리", "CVR", "CPA", "전환율" 등
   - 접속사/보조동사: "통해", "따라", "위해", "하기", "되어", "것으로" 등
   - 영어 불용어: "the", "is", "and", "very", "increase", "effect" 등

2. **강화된 메타 정보 필터링**
   - `[마케터 의도]`, `[타겟 심리]` 등 괄호 프레임 자동 제거
   - "전환율 10% 상승 예상", "CVR 2.5% 기대" 등 성과 지표 언급 제거
   - "...으로 판단됨", "...로 예상됨" 등 분석 결론 패턴 제거

3. **엄격한 키워드 품질 검증**
   - 최소 길이: 단일 키워드 4자 이상, 복합 키워드 6자 이상
   - 숫자 비율: 40% 이하만 허용 (기존 50%)
   - 한글/영문 비율: 60% 이상 필수
   - URL 패턴 자동 제외
   - 불용어 60% 이상 구성된 복합어 제외

4. **최종 필터링 강화**
   - 패턴 기반 분석 문구 제외: "예상됨", "판단됨", "보임" 등
   - 성과 메타 단어 제외: "전환율", "CVR", "지표", "수치" 등
   - 소재당 최대 5개 키워드 (기존 6개)로 품질 중시

**결과**: marketer_insight 필드에서 "와이드 뷰 거대 보스 대립", "레이드 콘텐츠 암시", "생존 압박감" 등 **구체적이고 실행 가능한 소재 특성 키워드**만 추출되어 군집화 정확도 향상
