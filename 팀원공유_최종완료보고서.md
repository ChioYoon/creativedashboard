## 📊 배포 가이드

### 🥇 **방법 1: GitHub Pages 배포 (권장)**

#### Step 1: GitHub 리포지토리 생성
```bash
1. GitHub 로그인 → New Repository
2. 리포지토리명: pepp-heroes-dashboard
3. Public 선택
4. Create repository
```

#### Step 2: 프로젝트 업로드
```bash
# 현재 프로젝트 폴더에서
git init
git add .
git commit -m "v1.7 배포 - 랜딩 페이지 + 사용 안내서 포함"
git branch -M main
git remote add origin https://github.com/yourcompany/pepp-heroes-dashboard.git
git push -u origin main
```

#### Step 3: GitHub Pages 활성화
```
1. GitHub 리포지토리 > Settings > Pages
2. Source: Deploy from a branch
3. Branch: main, Folder: / (root)
4. Save
5. 5분 후 URL 확인: https://yourcompany.github.io/pepp-heroes-dashboard/
```

## 🎯 사용 시나리오

### 시나리오 1: 신규 팀원 온보딩
```
1. 팀장이 Slack으로 URL 공유
2. 신규 팀원이 링크 클릭 → 랜딩 페이지 접속
3. "사용 가이드" 링크 클릭 → 사용안내서_인쇄용.html 확인
4. "🖨️ 인쇄하기" → PDF 저장 → 오프라인 참고
5. "대시보드 시작하기" 클릭 → step1_integrated.html 진입
6. CSV 업로드 → 점수 계산 → 결과 확인
```

### 시나리오 2: 주간 분석 루틴
```
1. 북마크에서 step1_integrated.html 바로 접속
2. CSV 업로드 (Google Ads 주간 리포트)
3. 가중치 조정 (필요 시)
4. 점수 계산 실행 (5분)
5. 최우수 소재 확인 → 예산 증액 후보 선정
6. 피로도 분석 실행
7. Tired 소재 확인 → 교체 계획 수립
8. 결과 스크린샷 → Slack 공유
```

### 시나리오 3: 팀 회의 활용
```
1. 노트북으로 step1_integrated.html 접속
2. 최신 CSV 업로드 후 점수 계산
3. 프로젝터로 결과 화면 공유
4. "📌 적용된 필터 조건" 확인 → 분석 범위 명확화
5. 최우수/개선필요 소재 논의
6. 피로도 차트 공유 → 순위 하락 소재 교체 결정
7. 차주 소재 제작 방향 합의
```
