## 🔄 v1.4 업데이트 (2026-04-08) - 전환수 기반 점수 계산!

### 주요 변경 사항
**문제**: 전환율과 IPM은 본질적으로 동일한 개념 (전환 / 노출수)  
**해결**: 전환율 점수 → **전환수 점수**로 변경하여 실제 전환 기여도 직접 반영

### 1. 📊 점수 계산 공식 변경

#### 변경 전 (v1.3):
```
전환율 점수 = (해당 소재 전환율 / 최대 전환율) × 100
Total Score = 전환율 점수 × 40% + CPA 점수 × 35% + IPM 점수 × 25%
```

#### 변경 후 (v1.4):
```
전환수 점수 = (해당 소재 전환수 / 전체 소재 전환수 합계) × 100
Total Score = 전환수 점수 × 40% + CPA 점수 × 35% + IPM 점수 × 25%
```

### 2. 🎯 개선 효과

| 측면 | 기존 (전환율) | 개선 (전환수) |
|-----|------------|-------------|
| **개념** | 상대적 효율성 | 절대적 기여도 |
| **IPM과의 관계** | 개념 중복 | 중복 제거 |
| **해석** | 효율성 비교 | 실제 가치 평가 |
| **합계** | 정규화 불가 | 항상 100% |
| **직관성** | 낮음 | 높음 |

## 🔧 v1.2 업데이트 (2026-04-08) - 중요!

### 핵심 문제 해결
**문제**: "Cannot read properties of undefined (reading 'length')" 오류 발생  
**원인**: `plParseCSV()` 함수가 `{ headers, rows }` 형식을 반환하는데, 코드에서는 `{ columns, rows }` 형식을 기대함  
**해결**: 반환값을 `{ columns: headers, rows }` 형식으로 변환

### 변경 사항
1. **CSV 파싱 로직 수정**
   - `plParseCSV(text)` → `{ headers, rows, _renameMap }` 반환
   - `plParseCSV`는 이미 `plNormalizeColumns`를 내부에서 호출하므로 중복 호출 제거
   - 반환값을 `normalizedData = { columns: headers, rows }` 형식으로 변환

2. **변수 스코프 문제 해결**
   - `campaigns` 변수를 try 블록 전체에서 사용할 수 있도록 수정
   - 업로드 상태 표시에서 `campaigns.length` 접근 오류 해결

3. **오류 메시지 개선**
   - CSV 파싱 실패 시 유효성 검사 추가
   - 스택 트레이스 콘솔 출력
   - 사용자에게 F12 콘솔 확인 안내

4. **테스트 파일 생성**
   - `test_step1_csv_upload.html`: 자동화된 CSV 업로드 테스트 (7단계)
   - 모든 테스트 통과 확인 ✅

## ⚙️ 핵심 기능

### Section 1: 📤 CSV 파일 업로드

#### 업로드 방법
1. **드래그 앤 드롭**: CSV 파일을 업로드 영역에 드래그
2. **파일 선택**: 업로드 영역 클릭 후 파일 브라우저에서 선택

#### 자동 처리
- **CSV 파싱**: `plParseCSV()` 함수로 CSV 텍스트를 객체 배열로 변환
- **컬럼 정규화**: `plNormalizeColumns()` 함수로 표준 컬럼명 매핑
- **캠페인 추출**: 중복 제거 후 정렬된 캠페인 목록 생성
- **날짜 범위 감지**: `detectDateColumn()` + `parseDate()` + `formatDate()`로 최소/최대 날짜 자동 설정
- **피로도 기간 분할**: 전체 기간을 전반부(기준)/후반부(비교)로 자동 분할

#### 업로드 상태 표시
```
✅ 업로드 완료
📁 파일명: PHPH.csv
📊 데이터: 997개 행, 41개 컬럼
🎯 캠페인: 3개
```

### Section 2: 📊 점수 계산 설정

#### 가중치 조정 (슬라이더)
- **전환율 가중치**: 0% ~ 100% (기본 40%)
- **CPA 가중치 (역수)**: 0% ~ 100% (기본 35%)
- **IPM 가중치**: 0% ~ 100% (기본 25%)
- **실시간 업데이트**: 슬라이더 조정 시 퍼센트 값 즉시 표시

#### 필터 옵션
- **분석 기준**: 
  - 소재명 (기본): 광고 카피 기준 집계
  - 파일명: 크리에이티브 파일 기준 집계
- **캠페인 필터**: 드롭다운에서 특정 캠페인 선택 (전체 캠페인 기본)
- **날짜 범위**: 시작일/종료일 지정 (자동 설정됨)

#### 실행 버튼
- **📊 점수 계산 실행**: 
  1. 필터 적용 (캠페인, 날짜, BNR/VID)
  2. 소재 집계 (전환, 비용, 노출수 합산)
  3. CPA, IPM, 전환율 계산
  4. 정규화 점수 산출 (최대값 기준 0~100)
  5. Total Score 계산 (가중 합산)
  6. 등급 부여 (최우수/우수/양호/보통/개선필요)
  7. 결과 표시 (통계 + 테이블)
- **🔄 설정 초기화**: 가중치, 분석 기준, 캠페인 필터 초기값 복원

function determineFatigueStatus(cpaChangeRate) {

### 1. 소재 집계 (aggregateCreatives)
```javascript
function aggregateCreatives(rows, groupBy) {
  const groups = {};
  
  rows.forEach(row => {
    const key = row[groupBy];  // '소재명' 또는 '파일명'
    if (!groups[key]) {
      groups[key] = { 전환: 0, 비용: 0, 노출수: 0, ... };
    }
    groups[key].전환 += parseFloat(row['전환']) || 0;
    groups[key].비용 += parseFloat(row['비용']) || 0;
    groups[key].노출수 += parseFloat(row['노출수']) || 0;
  });
  
  return Object.values(groups).map(g => {
    g.CPA = g.전환 > 0 ? g.비용 / g.전환 : 0;
    g.IPM = g.노출수 > 0 ? (g.전환 / g.노출수) * 1000 : 0;
    g.전환율 = g.노출수 > 0 ? (g.전환 / g.노출수) * 100 : 0;
    return g;
  });
}
```

### 2. 점수 계산 (calculateCreativeScores)
```javascript
function calculateCreativeScores(creatives, convWeight, cpaWeight, ipmWeight) {
  // 1. 최대값 찾기
  const maxConv = Math.max(...creatives.map(c => c.전환율));
  const maxCPA = Math.max(...creatives.map(c => c.CPA));
  const maxIPM = Math.max(...creatives.map(c => c.IPM));
  
  // 2. 정규화 점수 산출 (0~100)
  creatives.forEach(c => {
    c.전환율점수 = maxConv > 0 ? (c.전환율 / maxConv) * 100 : 0;
    c.CPA점수 = maxCPA > 0 ? (1 - (c.CPA / maxCPA)) * 100 : 0;  // 역수
    c.IPM점수 = maxIPM > 0 ? (c.IPM / maxIPM) * 100 : 0;
    
    // 3. Total Score (가중 합산)
    c.TotalScore = 
      c.전환율점수 * convWeight +
      c.CPA점수 * cpaWeight +
      c.IPM점수 * ipmWeight;
  });
  
  // 4. 정렬 (Total Score 내림차순)
  creatives.sort((a, b) => b.TotalScore - a.TotalScore);
  
  // 5. 등급 부여
  creatives.forEach(c => {
    if (c.TotalScore >= 80) c.등급 = '최우수';
    else if (c.TotalScore >= 60) c.등급 = '우수';
    else if (c.TotalScore >= 40) c.등급 = '양호';
    else if (c.TotalScore >= 20) c.등급 = '보통';
    else c.등급 = '개선필요';
  });
  
  return creatives;
}
```

### 3. 피로도 비교 (compareFatigue)
```javascript
function compareFatigue(baseline, comparison) {
  // 1. 순위 부여 (CPA 오름차순)
  baseline.sort((a, b) => a.CPA - b.CPA);
  baseline.forEach((c, i) => c.rank = i + 1);
  
  comparison.sort((a, b) => a.CPA - b.CPA);
  comparison.forEach((c, i) => c.rank = i + 1);
  
  // 2. Map 생성
  const baseMap = {};
  baseline.forEach(c => baseMap[c.key] = c);
  
  const results = [];
  
  // 3. 비교 기간 소재 처리
  comparison.forEach(compItem => {
    const baseItem = baseMap[compItem.key];
    
    if (baseItem) {
      // 기존 소재
      const rankChange = baseItem.rank - compItem.rank;
      const cpaChange = compItem.CPA - baseItem.CPA;
      const cpaChangeRate = baseItem.CPA > 0 ? (cpaChange / baseItem.CPA) * 100 : 0;
      
      // 피로도 판정
      let fatigueStatus = 'stable';
      if (cpaChangeRate <= -20) fatigueStatus = 'fresh';
      else if (cpaChangeRate > 0 && cpaChangeRate <= 20) fatigueStatus = 'warning';
      else if (cpaChangeRate > 20) fatigueStatus = 'tired';
      
      results.push({
        key: compItem.key,
        baselineRank: baseItem.rank,
        comparisonRank: compItem.rank,
        rankChange: rankChange,
        cpaChangeRate: cpaChangeRate,
        fatigueStatus: fatigueStatus,
        ...
      });
    } else {
      // 신규 소재
      results.push({
        key: compItem.key,
        fatigueStatus: 'new',
        ...
      });
    }
  });
  
  // 4. 사라진 소재 처리
  baseline.forEach(baseItem => {
    if (!compMap[baseItem.key]) {
      results.push({
        key: baseItem.key,
        fatigueStatus: 'disappeared',
        ...
      });
    }
  });
  
  // 5. 정렬 (기준 순위 오름차순)
  results.sort((a, b) => {
    if (a.baselineRank === '-') return 1;
    if (b.baselineRank === '-') return -1;
    return a.baselineRank - b.baselineRank;
  });
  
  return results;
}
```

## 💼 비즈니스 활용

### Use Case 1: 주간 성과 리뷰
**시나리오**: 매주 월요일, 지난주 광고 소재 성과 분석
1. CSV 업로드 (지난주 데이터)
2. 점수 계산 실행 → 상위 10개 소재 확인
3. 피로도 분석 실행 (월~수 vs 목~일) → 주중/주말 비교
4. 🔴 피로 소재 → 교체 목록 작성
5. 🟢 신선 소재 → 예산 증액 제안

### Use Case 2: 캠페인별 A/B 테스트
**시나리오**: 2개 캠페인의 소재 성과 비교
1. CSV 업로드 (전체 캠페인 데이터)
2. 캠페인 필터 = "캠페인 A" → 점수 계산
3. 캠페인 필터 = "캠페인 B" → 점수 계산
4. 두 캠페인의 평균 점수, 최우수 소재 수 비교
5. 우수한 캠페인의 크리에이티브 전략 벤치마크

### Use Case 3: 월간 피로도 트렌드
**시나리오**: 한 달간 소재 피로도 변화 추적
1. CSV 업로드 (월간 데이터)
2. 1주차 vs 4주차 피로도 분석
3. 순위 변동 차트에서 장기 트렌드 확인
4. CPA 변화율 상위/하위 10개 소재 리스트업
5. 피로 소재 리프레시 주기 결정 (2주? 3주?)

### Use Case 4: 소재명 vs 파일명 비교
**시나리오**: 동일 소재명, 다른 크리에이티브 파일의 성과 차이
1. 분석 기준 = "소재명" → 점수 계산 (23개 소재)
2. 분석 기준 = "파일명" → 점수 계산 (59개 파일)
3. 동일 소재명 내에서 파일별 Total Score 비교
4. 저성과 파일 → 디자인 개선 방향 도출
5. 고성과 파일 → 크리에이티브 가이드 작성
