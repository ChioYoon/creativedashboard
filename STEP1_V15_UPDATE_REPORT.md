## ✅ 완료된 작업

### 1. ROAS 지표 추가 📊
- **Revenue 컬럼 자동 감지**: `매출`, `Revenue`, `revenue`, `REVENUE` 컬럼 자동 인식
- **ROAS 계산**: ROAS = 매출 / 비용
- **ROAS 점수**: Rank 기반 정규화 (ROAS가 높을수록 높은 점수)
- **UI 업데이트**:
  - 테이블에 ROAS 컬럼 추가 (ROAS > 1이면 녹색, ≤ 1이면 빨간색)
  - 모달에 ROAS 및 매출 정보 표시
  - 가중치 슬라이더에 ROAS 가중치 추가

### 2. Rank 기반 점수 계산으로 통일 🏆
**기존 (v1.4)**: 전환수 점수만 비율 기반 (전환수 / 전체 합계 × 100)

**변경 (v1.5)**: 모든 지표를 Rank 기반으로 통일

#### 새로운 점수 계산 방식:
```javascript
// Step 1: 각 지표별 Rank 부여
전환수 Rank: 전환수 내림차순 정렬 (많을수록 1위)
CPA Rank: CPA 오름차순 정렬 (낮을수록 1위)
IPM Rank: IPM 내림차순 정렬 (높을수록 1위)
ROAS Rank: ROAS 내림차순 정렬 (높을수록 1위)

// Step 2: Rank를 점수로 변환
점수 = (n - Rank + 1) / n × 100

// 예: 4개 소재 중 1위 → (4-1+1)/4 × 100 = 100점
//     4개 소재 중 4위 → (4-4+1)/4 × 100 = 25점

// Step 3: 가중 평균
Total Score = 
  전환수 점수 × 25% +
  CPA 점수 × 25% +
  IPM 점수 × 25% +
  ROAS 점수 × 25%
```

#### Rank 기반 점수의 장점:
- ✅ **이상치 영향 최소화**: 극단적인 값의 영향 감소
- ✅ **공정한 비교**: 모든 지표가 동일한 스케일 (0-100)
- ✅ **상대적 순위 명확**: 소재 간 상대적 우위가 명확히 드러남
- ✅ **일관성**: 모든 지표가 동일한 방식으로 계산됨

### 3. 캠페인 다중 선택 필터 🎯
- **HTML**: `<select multiple size="4">` 적용
- **사용법**: Ctrl/Cmd + 클릭으로 여러 캠페인 선택
- **로직**: 
  - 배열로 선택된 캠페인 저장
  - 필터링 시 `includes()` 메서드 사용
  - "전체 캠페인" 선택 시 필터 비활성화
- **적용 범위**:
  - 점수 계산 섹션의 캠페인 필터
  - 피로도 분석 섹션의 캠페인 필터

### 4. 가중치 직접 입력 기능 ⌨️
- **UI 구성**: 슬라이더 + 숫자 입력 필드 + 퍼센트 표시
- **양방향 동기화**:
  - 슬라이더 변경 → 입력 필드 & 표시 업데이트
  - 입력 필드 변경 → 슬라이더 & 표시 업데이트
- **범위 검증**: 0~100 자동 제한
- **4개 가중치**:
  - 전환수 가중치 (기본 25%)
  - CPA 가중치 (기본 25%)
  - IPM 가중치 (기본 25%)
  - ROAS 가중치 (기본 25%)

### ROAS 기준 해석

| ROAS | 의미 | 액션 |
|------|-----|-----|
| **> 3.0** | 매출이 비용의 3배 이상 | 🟢 핵심 소재 - 예산 증액 |
| **2.0 ~ 3.0** | 매출이 비용의 2~3배 | 🔵 우수 소재 - 현상 유지 |
